#+TITLE: NixOS Configuration
#+AUTHOR: aRenCoco

* 简介
这是一个基于 Flake 的 NixOS 配置仓库.

* 结构

** [[file:./][/]]
#+begin_example
/
├── .gitignore
├── LICENSE
├── README.org
├── flake.lock
├── flake.nix
└── modules/

仓库根目录放置 flake 入口, lock 文件, README 和模块目录. =flake.nix= 使用 =flake-parts= 组织输出, 定义 =nixosConfigurations= 和 =nixos-anywhere= flake app. =nixpkgs= 使用 =nixos-unstable=, 并启用 =nix-command= 与 =flakes=.

[[file:./modules/features/default.nix][modules/features/default.nix]] 按功能领域组织 NixOS 与 home-manager 模块, 自动发现含 =default.nix= 的 feature 目录, 定义 =modules.features.*= option, 并把启用的 HM feature 导入到对应用户.

所有 feature 都由 [[file:./modules/features/default.nix][modules/features/default.nix]] 自动发现, 并以 =modules.features.<name>= option 暴露. 各 host 通过 =modules.features.<name>.enable= 按需启用 feature; 含 HM 模块的 feature 会按 =allowUidList= 导入到对应用户.

每个 feature 以 =default.nix= 作为入口, 并按 =os/= (系统级), =hm/= (用户级), =lib/= (模块内部函数) 分层. 下列 feature 小节会把这些内部子目录放在各自代码块与说明中, 不再单独展开为 headline.

***** [[file:./modules/users/lingyu/][/modules/users/lingyu]]
#+begin_example
/modules/users/lingyu/
├── default.nix
├── cloud-server.nix
├── email.nix
├── git.nix
├── home.nix
├── keyring.nix
├── langs.nix
├── terminal.nix
└── .local/
    └── bin/
        ├── org.sh
        ├── rc.sh
        └── ssha.sh
#+end_example

配置 lingyu 用户的个人 home-manager 模块, 包含 email, git, keyring, langs, terminal 等用户级配置.

=modules/users/lingyu/.local/bin/= 放置个人脚本:
- =org=: 进入 =~/org= 目录并准备 make 函数, 支持 --texlive 调用 TeX Live 工具.
- =rc=: rclone 同步包装器, 支持 push/pull 子命令, 限制传输并发并显示进度.
- =ssha=: 启动 =ssh-agent= 并加载默认 ed25519 私钥.
